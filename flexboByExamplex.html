Flex Container properties

Here the flex container elements (the red border wrapping the blue boxes) have the display: flex;.

They also have one of the following to give them extra styles

The flex-direction

This property defines in which direction the container wants to stack the flex items. Default value is flex-direction: row;
So if you leave that out, it will still look like this.

Its worth noting there is a "flex-direction: collumn" property that can be used to rotate all the flexbox operations 90 degrees. However its much easier to learn everything horizontally and then flip with flex-direction if you need to. Rather than trying to learn the way everything works on two axis.

The justify-content property (items in row horizontally)

is used to Horizontally align the flex items. In flexbox, the word "justify" signals horizotal adjustments, and the word "align" signals vertical adjustments.
Here we use justify-content: flex-end;
Values: center, flex-start, flex-end, space-around, space-between, space-evenly;

The align-items (manipulate all items vertically in row) property is used to align all items in the flex row vertically

Values: center, flex-start, flex-end, stretch, baseline.
TIP: the difference between align-items: center; and align-content: center;
  1. align-items does not wrap by default, the items will just shrink.
  2. align-content does wrap if the viewport becomes too small.

TIP: Because align-items is vertical. Setting a height for the stretch property wont do anything.

The "align-items: baseline;" aligns the flex items by their baseline. Here using h1 h3 h3 and small elements to illustrate different baselines by element.

Child Elements (Items)

The direct child elements of a flex container (the blue boxes inside the red borders) automatically becomes flexible (flex) items.

You can specify display: flex; on the blue boxes, but that just makes them into their own flex containers. For these three properties to work they must be wrapped by something with the display: flex; property.

The flex item properties are:
  1. order
  2. flex-grow
  3. flex-shrink
  4. flex-basis
  5. flex
  6. self-align

flexbox order property

The order property specifies the order of the flex items. Here the display order is backwards from the order these elements are written in the html.

flex-grow

The value must be a number.
Default value is 0.
basically, if you are stretching the page, everything will expand, but the elements with this value will expand faster or slower proportionally (based on the space available to expand into) to each other. the Default value is 0.
If you add more items to the row the growth will be less pronounced
NOTE: You might have to shrink the page to see the effect!

flex-shrink

The flex-shrink property specifies how much a flex item will shrink relative to the rest of the flex items.
The value must be a number,default value is 1.
basically, if you are shrinking the page, everything will shrink, but the elements with this value will shrink faster or slower proportionally to each other.
The Default value is 0.
The following will shrink faster from the right than the left.

flex-basis

The flex-basis property specifies the initial length horizontally of a flex item.
I belie this means, initial size of an item that will be changed by the flex-shrink or flex-grow properties in the screen size is changed
This is complex, please refer to the documentation for This https://developer.mozilla.org/en-US/docs/Web/CSS/flex-basis as near as I can tell, this just sets the default width of the item. But not reliably.
Its very very hard to find any examples that actually show what you would ever want to use this for.

flex-basis: auto;

flex property shorthand

The flex property is a shorthand property for the flex-grow,flex-shrink, and flex-basis properties. 
flex-grow default is 0. 0 means no growth. a 1 will fill all space left by other 0's. a 2 will simply fill twice as much space as a 1 and so on.
flex-shrink default is 1. 0 will not shrink at all. 1 shrinks normally. Higher numbers shrink faster
Here each of these has a different set of values. But i am leaving the flex basis at auto for each.
Highly recommend playing with the screen width to see how this works in real time. You might have to shrink very small to see the shrink in action.
flex: 2 0 auto
flex: 1 0 auto
flex: 0 0 auto
flex: 0 1 auto; width: 100px;
flex: 0 2 auto; width: 100px;
flex: 0 3 auto; width: 100px;

You can leave out up to two values for the flex property

It will intelligently assume what you mean, defaulting to flex-grow with one number. Or to flex-basis with a word or a size.
